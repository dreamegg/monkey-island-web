## Project Overview

A SCUMM-style point-and-click adventure game engine for the web, inspired by *The Secret of Monkey Island* (1990). The game is in Korean (UI text, dialogues, object descriptions). Built with React 18 + TypeScript + Vite, rendering via HTML5 Canvas with `image-rendering: pixelated`.

## Commands

```bash
cd monkey-island-web
npm install
npm run dev          # Start Vite dev server with HMR
npm run build        # TypeScript check + Vite production build
npm run preview      # Preview production build locally
npm run deploy       # Build + deploy to GitHub Pages via gh-pages
```

No test framework is configured yet. No linter is configured.

## Architecture

The project has two layers: a standalone JSX prototype (`scumm-engine.jsx` at root) and the main TypeScript app (`monkey-island-web/`).

### State Management

Central game state lives in a **Zustand store** (`src/engine/GameEngine.ts` — `useGameStore`). All game actions (movement, verb interactions, dialogue, inventory, room transitions, flags) are store actions. Components subscribe to individual slices via selectors.

### Engine Modules (src/engine/)

- **GameEngine.ts** — Zustand store: game state + all actions (movement ticks, verb execution, dialogue flow, room changes, flag system). This is the single source of truth.
- **CanvasRenderer.ts** — Renders the scene each frame: tries to load a background image from `/assets/backgrounds/{roomId}.png`, falls back to procedural room renderers. Draws NPCs (sprite or procedural), hovered-object highlights, and the player character.
- **SceneManager.ts** — Hit-testing and hover detection. Maps canvas pixel coordinates to objects/exits/NPCs using their normalized position + pixel dimensions.
- **VerbSystem.ts** — Executes verb+object actions. Looks up `obj.actions[verb]`, handles `pick_up` with item acquisition, falls back to random Korean failure messages.
- **DialogueEngine.ts** — Tree-based dialogue system with `DialogueTree`/`DialogueNode`/`DialogueChoice`. Supports conditional choices (`condition`), one-time choices (`once`), and `onEnter` callbacks for side effects (setting flags, giving items).
- **Pathfinding.ts** — Walk area clamping. Supports polygon walk areas (from depth analysis) with rectangle fallback.
- **DepthSystem.ts** — Runtime perspective system. Loads depth maps + room configs generated by `tools/depth-analyzer/`. Provides per-pixel depth lookup for character scaling and polygon-based walkability checks. All functions gracefully fall back when no depth config exists.
- **types.ts** — All shared types, constants (`CANVAS_W=1600`, `CANVAS_H=800`), color `PALETTE`, and `VERBS` array.

### Coordinate System

Objects and positions use **normalized coordinates** (0-1 range) for x/y, but **pixel dimensions** for width/height (w/h). The canvas is 1600x800. Hit-testing multiplies normalized positions by CANVAS_W/CANVAS_H but uses raw pixel w/h values. This mixed system is important to understand when adding new objects or rooms.

### Room Data (src/data/rooms.ts)

Rooms are defined as TypeScript objects with procedural `render()` functions that draw backgrounds directly to canvas. Each room has: walkArea (normalized rect), objects (with verb actions), exits (with walkTo targets), and optional NPCs. Current rooms: harbor, tavern, forest, beach, cave.

### Dialogue Data (src/data/dialogues.ts)

Dialogue trees are registered at startup via `initDialogues()`. They use `onEnter` callbacks to trigger side effects (flags, items). Flag-gated choices (e.g., `knows_trials`) control progressive story unlocks.

### Asset Loading Strategy

The renderer tries image assets first, falls back to procedural drawing. This allows incremental asset replacement:
- Backgrounds: `public/assets/backgrounds/{roomId}.png`
- Sprites: `public/assets/sprites/{name}_idle.png`
- Portraits: `public/assets/portraits/{name}.png`
- Audio: loaded via `src/utils/audioManager.ts` using Howler.js

### Components (src/components/)

- **GameCanvas.tsx** — Canvas element, animation/movement tick loops, WASD/arrow key input, click/hover handlers
- **IntroScene.tsx** — Scripted intro cutscene with typewriter effect, portrait dialogue, skip support
- **VerbPanel.tsx** — 3x3 grid of SCUMM verbs
- **InventoryBar.tsx** — Inventory display panel
- **DialogueBox.tsx** — Overlay dialogue UI with choices
- **MessageBar.tsx** — Bottom message display

### Depth / Perspective System

Character and NPC sprites scale based on their Y position using AI-generated depth maps. The pipeline:
1. Run `tools/depth-analyzer/analyze.py` on background images (uses Depth Anything V2)
2. Generates per-room: depth map PNG (`public/room-configs/{roomId}_depth.png`), config JSON, and debug visualization
3. Engine loads these at startup via `preloadAllDepthConfigs()` and applies scaling in `CanvasRenderer.ts`
4. Walk area polygons extracted from depth analysis replace the legacy rectangle walkboxes

When no depth config exists for a room, everything falls back to the previous behavior (no scaling, rectangle walk area).

### Dev Tools

A separate web page at `/devtools.html` (separate Vite entry point) for game development:
- **Room Editor** — Canvas preview with toggleable overlays: walk area polygon, object hotspots, exit regions, NPC positions, depth scale preview, coordinate grid. Shows mouse position and scale factor in real-time.
- **Dialogue Viewer** — SVG flow chart of dialogue trees with node details, choice conditions, and flag side-effects.
- **Items & Inventory** — Table of all obtainable items, their source (object pickup vs dialogue), and game flags.
- **Room Map** — Graph visualization of room transitions with summary stats.

### Tools

- `tools/depth-analyzer/` — Python script using Depth Anything V2 for automatic depth map + walk area generation from background images. Requires torch, transformers, opencv-python.
- `tools/asset-pipeline/` — Python scripts for FLUX.1 + PixelArt LoRA image generation and post-processing
- `tools/audio-pipeline/` — Audio asset generation

## Key Design Decisions

- **Korean-language game**: All in-game text, dialogue, and UI labels are in Korean. Maintain this when adding content.
- **Procedural-first rendering**: Rooms draw backgrounds via Canvas API calls. Image assets are optional overlays, not required.
- **Depth-based perspective**: Character/NPC sprites scale by depth map lookup. Depth configs are optional — engine falls back gracefully.
- **Normalized positions**: Player and object positions use 0-1 range relative to canvas dimensions. Object w/h are in absolute pixels.
- **No ScriptRunner yet**: The PRD defines a JSON script command system (say, walk, anim, set_flag, etc.) but it's not implemented. Verb actions currently return static strings.
- **Phase 1 complete, Phase 2 in progress**: Core engine works (movement, verbs, inventory, rooms, dialogue). TypeScript migration done. Zustand integrated. Remaining: JSON room loader, script runner, save/load, item combinations.
